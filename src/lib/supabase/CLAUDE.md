# Supabase — `src/lib/supabase/`

Database and auth clients. Get the client choice wrong and you either break RLS
or bypass it entirely.

## Three clients — which to use

| Client | Where | RLS | Use for |
|---|---|---|---|
| `createBrowserClient()` / `supabase` (default export) | Client components, hooks | Enforced | All client-side reads/writes |
| `createSupabaseApiClient(req, res)` (in `@/lib/cfp/auth`) | API routes | Enforced | Any handler acting as a logged-in user |
| `createSupabaseServerClient(ctx)` (in `@/lib/cfp/auth`) | `getServerSideProps` | Enforced | SSR with cookie-based session |
| `createServiceRoleClient()` | Webhooks, admin-only system jobs | **Bypassed** | Stripe webhook, system backfills, never user-context |
| `createCfpServiceClient()` (in `cfp-client.ts`) | CFP-specific service-role ops | Bypassed | CFP admin tooling that needs to ignore RLS |

## Service-role rules

Importing `createServiceRoleClient` should make you stop and ask: "Why am I bypassing RLS?"

Valid answers:
- Stripe webhook — request is signed, not user-authenticated.
- Migration / backfill scripts.
- Admin endpoints that need cross-user visibility AND have their own auth check via `verifyAdminAccess`.

Invalid answers:
- "It's easier than figuring out the RLS policy" — fix the policy.
- "The user is logged in" — use `createSupabaseApiClient` instead.

## Generated types

`src/lib/types/database.generated.ts` is the source of truth for table shapes. It's
generated by `scripts/regen-db-types.sh` from the live Supabase schema. **Never
hand-edit it.**

Clients are typed with `Database` from `@/lib/types/database` (the wrapper around
the generated file).

## Cookie-based session sync

Server-side clients (`createSupabaseApiClient`, `createSupabaseServerClient`) use
`@supabase/ssr` to read/write the session via cookies. This is how a logged-in user
in the browser is recognized by an API route. Don't roll your own auth header
plumbing.

## Singleton browser client

`createBrowserClient()` returns a cached singleton — multiple calls reuse the same
instance. The default `supabase` export is that singleton.
